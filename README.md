# 🏥 Hospital Patient Flow & Resource Management System (SQL Project)

## 📌 Project Overview

This project demonstrates an advanced **SQL-based data analysis system** for managing hospital operations, focusing on **patient flow, resource utilization, and financial performance**.

The system simulates real-world hospital scenarios including:

* Patient admissions & discharges
* Doctor assignments & workload
* Bed and ward management
* Surgery tracking
* Billing and insurance processing

---

## 🎯 Objectives

* Analyze **hospital operational efficiency**
* Monitor **patient flow & treatment lifecycle**
* Optimize **bed and resource utilization**
* Evaluate **doctor performance**
* Track **financial metrics (billing, insurance, revenue)**

---

## 🗂️ Database Schema

### Core Tables:

* **Patients** – Patient demographics, insurance, medical history
* **Doctors** – Doctor details, specialization, department
* **Beds** – Bed allocation, ward types, availability
* **Admissions** – Patient admission records, diagnosis, discharge
* **Surgeries** – Surgical procedures, duration, outcomes
* **Billing** – Financial transactions, insurance claims, payments

---

## 🔗 Entity Relationships

* One **Patient** → Multiple **Admissions**
* One **Doctor** → Multiple **Admissions & Surgeries**
* One **Admission** → One **Bed** & One **Billing Record**
* One **Admission** → Multiple **Surgeries**

---

## 📊 Key SQL Features Used

* Complex **JOINs** (INNER, LEFT)
* **Window Functions** (LAG, LEAD, RANK)
* **CTEs (Common Table Expressions)**
* **Aggregations & Grouping**
* **Subqueries & Nested Queries**
* **Date Functions & Time Analysis**
* **Performance Optimization Techniques**
